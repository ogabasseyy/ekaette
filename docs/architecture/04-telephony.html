2. Split Telephony Control Plane + Shared AI Brain

The reference deployment keeps one ADK agent brain for every channel, but deliberately separates short-lived HTTP control traffic from long-lived voice media streams.

Main HTTP service: webhooks, XML call control, text channels, callback orchestration.
Live voice service: long-lived /ws audio streams and Runner.run_live() sessions.
SIP bridge VM: codec conversion, denoise, callback prewarm, explicit SIP BYE.

3. Inbound Phone Call — AT Control Plane + SIP Media Plane

sequenceDiagram
    participant CALLER as Caller (Phone)
    participant ATC as Africa's Talking
(voice callback)
    participant MAIN as Cloud Run main service
(/api/v1/at/voice/callback)
    participant ATS as Africa's Talking
(SIP registrar)
    participant SIP as SIPServer
(GCE VM :6060)
    participant SESS as CallSession
(4-task pipeline)
    participant GW as GatewayClient
(WSS)
    participant LIVE as Cloud Run live voice service
(ADK voice runtime)

    Note over CALLER,LIVE: SIP Registration (on startup)
    SIP->>ATS: SIP REGISTER (UDP)
    ATS->>SIP: 401 Unauthorized (nonce challenge)
    SIP->>SIP: Compute digest auth (MD5/SHA-256)
    SIP->>ATS: SIP REGISTER (with Authorization)
    ATS->>SIP: 200 OK (registered)
    Note over SIP,ATS: Re-registers at 80% of expiry

    Note over CALLER,LIVE: Inbound Call
    CALLER->>ATC: Dial service number
    ATC->>MAIN: POST /api/v1/at/voice/callback (isActive=1)
    MAIN-->>ATC: XML Dial sip:...
    ATC->>ATS: Execute Dial to SIP endpoint
    ATS->>SIP: SIP INVITE (SDP: G.711, port X)
    SIP->>ATS: 100 Trying
    SIP->>SIP: Allocate RTP port, extract caller phone from From header
    SIP->>SIP: Derive namespaced gateway IDs from tenant/company + caller/call context
    SIP->>SESS: Create CallSession + GatewayClient

    Note over SIP,LIVE: Live voice path comes up before the AT leg is answered
    SESS->>GW: connect() → WSS to live voice service /ws/{user_id}/{session_id}?token=...
    GW->>LIVE: WebSocket handshake
    LIVE-->>GW: session_started {sessionId}
    SIP->>ATS: 200 OK (SDP: G.711, port Y)
    ATS->>SIP: ACK

    Note over CALLER,LIVE: Bidirectional Audio (4 concurrent tasks)

    par Caller speaks
        CALLER->>ATC: Voice audio
        ATS->>SIP: RTP G.711 frames
        SESS->>SESS: G.711 decode → resample 8k→16k
→ denoise / gain normalization
        SESS->>GW: send_audio(PCM16 16kHz)
        GW->>LIVE: Binary WebSocket frame
    and AI responds
        LIVE->>GW: Binary WebSocket frame (PCM16 24kHz)
        GW->>SESS: GatewayFrame(audio)
        SESS->>SESS: Resample 24k→8k→G.711→RTP
        SESS->>ATS: RTP audio frames
        ATS->>CALLER: Caller hears AI
    end

    Note over CALLER,LIVE: Callback legs can end with an explicit bridge-originated SIP BYE after the acknowledgement and audio drain
    ATS->>SIP: SIP BYE or bridge-originated BYE
    GW->>LIVE: Close WebSocket

4. Inbound Phone Call — Fallback Direct Mode (Hardcoded Prompt)

    GEMINI-->>SESS: WebSocket connected
    SESS->>GEMINI: send_client_content("[Call connected]")
    GEMINI->>SESS: Audio greeting (PCM16 24kHz)

    par Caller speaks
        CALLER->>AT: Voice audio
        AT->>SESS: RTP G.711 frames
        SESS->>GEMINI: send_realtime_input(PCM16 16kHz)
    and AI responds
        GEMINI->>SESS: PCM16 24kHz chunks
        SESS->>AT: RTP audio frames
        AT->>CALLER: Caller hears AI
    end

    Note over CALLER,GEMINI: Call Ends
    SESS->>GEMINI: Close WebSocket

Note: Phone CallSession does not write Firestore call records (unlike WaSession which persists to wa_calls).
